# 下面进行单文件搜索的实现

## 先给你整体思路

这一步你会学会 4 件事：

- **怎么把命令行参数变成配置对象**
- **怎么封装匹配器 `Matcher`**
- **怎么逐行读取文件**
- **怎么把搜索结果组织成结构体返回**

这一步很重要，因为后面做目录并发搜索时，**单文件搜索逻辑基本不用推倒重来**。



# 第 2 个文件：`internal/model/result.go`

写成这样：

go





```
package model
 
type LineMatch struct {
    LineNum int
    Content string
}
 
type FileMatch struct {
    FilePath string
    Lines    []LineMatch
}
```

------

## 为什么要先建结果结构

因为你不要把“搜索结果”只当成一堆打印语句。
你要把它建模成数据。

这样以后你才能：

- 终端打印
- JSON 输出
- 统计匹配数量
- 做上下文行
- 做替换功能



# 这部分你一定要理解（对于Macher.go)

## 为什么要单独做 `Matcher`

因为“如何匹配”是独立逻辑。

搜索文件的时候你只关心：

- 这一行匹不匹配？

至于是：

- 正则匹配
- 固定字符串匹配
- 忽略大小写

这些细节都应该藏在 `Matcher` 里。

这就是封装。



### `bufio.NewScanner(file)`

这是逐行读取文件最方便的方式。

你这里做的是：

- 不一次把整个文件读进内存
- 而是一行一行地读

这对后面大文件搜索特别重要
